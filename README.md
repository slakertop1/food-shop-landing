# 🛒 Лендинг интернет-магазина с корзиной и онлайн-оплатой

Одностраничный магазин под быстрый запуск продаж ограниченной партии товара:
каталог, корзина, оформление заказа, **онлайн-оплата через ЮKassa** и приём
заказов на **email + Telegram**. Без личного кабинета, без интеграций с 1С —
ровно то, что нужно, чтобы начать продавать за пару недель.

Стек: **Python · FastAPI · SQLite · чистые HTML/CSS/JS** (без конструкторов и
фронтенд-фреймворков). Разворачивается на любом VPS одной командой.

> **Демо-режим оплаты «из коробки».** Без ключей ЮKassa магазин работает в
> демо-режиме: платёжная страница имитируется, деньги не списываются — весь
> путь заказа можно показать до подключения боевого эквайринга.

## Что умеет

- 🧺 **Каталог** (20 SKU) с категориями, ценами и количеством за единицу.
- 🛒 **Корзина** с сохранением между визитами (localStorage), изменением
  количества прямо в карточке товара и на панели корзины.
- 📦 **Минимальная сумма заказа** — кнопка оформления блокируется, пока сумма
  не набрана, с подсказкой «добавьте ещё на N ₽».
- 📝 **Оформление**: ФИО, телефон, адрес, комментарий, выбор способа оплаты
  (онлайн на сайте / наличными-картой курьеру).
- 💳 **Онлайн-оплата ЮKassa**: создание платежа, редирект на оплату,
  подтверждение по вебхуку. Демо-режим, когда ключи не заданы.
- 📨 **Уведомления о заказе** на email (SMTP) и в Telegram.
- 🚫 **Переключатель «продажи приостановлены»** — одна команда, и на сайте
  появляется баннер, а оформление отключается. Для распроданной партии.
- 🔒 **Цены считаются на сервере** по каталогу — подделать стоимость из браузера
  нельзя (частая уязвимость самописных магазинов).

## Быстрый старт

```bash
cp .env.example .env      # заполните при необходимости (можно оставить как есть — будет демо)
```

### Docker

```bash
docker compose up -d --build
# магазин на http://localhost:8000
```

### Вручную

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Откройте `http://localhost:8000`, соберите корзину, оформите заказ — в демо-режиме
оплата пройдёт через страницу-заглушку, а заказ упадёт в лог (и в Telegram/email,
если настроены).

## Подключение оплаты (ЮKassa)

1. Зарегистрируйте магазин в [ЮKassa](https://yookassa.ru) (нужно ИП или ООО).
2. В личном кабинете: **Настройки → Ключи API** — скопируйте `shopId` и секретный ключ.
3. Впишите их в `.env` (`YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY`) и укажите
   реальный `BASE_URL` (адрес вашего сайта с https).
4. В ЮKassa настройте **HTTP-уведомления** на `https://ваш-домен/api/yookassa/webhook`
   (событие `payment.succeeded`).

Как только ключи заданы — демо-режим выключается автоматически, оплата идёт через
настоящую платёжную форму ЮKassa.

## Управление продажами

Включить/выключить приём заказов (например, когда партия закончилась):

```bash
# выключить
curl -X POST http://localhost:8000/api/admin/sales \
  -H "X-Admin-Token: ВАШ_ADMIN_TOKEN" -H "Content-Type: application/json" \
  -d '{"enabled": false}'

# включить обратно
curl -X POST http://localhost:8000/api/admin/sales \
  -H "X-Admin-Token: ВАШ_ADMIN_TOKEN" -H "Content-Type: application/json" \
  -d '{"enabled": true}'
```

Токен задаётся в `.env` (`ADMIN_TOKEN`). При выключенных продажах на сайте
показывается баннер, а оформление заказа блокируется на сервере.

## Каталог товаров

20 товаров лежат в [`data/products.json`](data/products.json) — правится вручную
или выгружается из вашей таблицы. Поля: `id`, `name`, `cat` (категория), `price`,
`unit` (фасовка), `emoji` (заглушка вместо фото — меняется на реальные картинки),
`desc`. Для боевого магазина эмодзи заменяются на фотографии товаров.

## Настройки (.env)

| Переменная | Описание |
|---|---|
| `SHOP_NAME` | название магазина |
| `MIN_ORDER` | минимальная сумма заказа, ₽ |
| `BASE_URL` | публичный адрес сайта (для оплаты и вебхука) |
| `YOOKASSA_SHOP_ID` / `YOOKASSA_SECRET_KEY` | ключи эквайринга (пусто → демо) |
| `ORDER_EMAIL`, `SMTP_*` | куда и через какой SMTP слать заказы |
| `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` | дубль заказа в Telegram |
| `ADMIN_TOKEN` | токен управления продажами |

